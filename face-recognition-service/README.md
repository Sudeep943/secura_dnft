# Secura Face Recognition Microservice

A lightweight Python microservice that extracts **128-dimensional face embeddings** from images.  
It uses [face_recognition](https://github.com/ageitgey/face_recognition) (backed by dlib's ResNet model) which analyses **68 facial landmarks** — eyes, nose, mouth, eyebrows, and jawline — to produce a consistent descriptor for the same person across different photos, lighting conditions, and moderate pose variation.

---

## Why This Is Required

The Spring Boot backend ships with a SHA-256 hash **stub** that produces an embedding purely from the raw image bytes. That stub is for integration testing only (identical bytes → identical embedding). It **cannot** match two different photos of the same person, which is why "face not recognized" errors occur in real usage.

Configuring this service replaces the stub with genuine face recognition.

## Quick Start (local development)

```bash
# 1. Install Python dependencies (Python 3.9+ required)
pip install -r requirements.txt

# 2. Start the service
python app.py
# Listening on http://localhost:5001
```

## API

### `POST /extract-embedding`

Extracts a 128-dimensional face embedding.

**Request body (JSON):**
```json
{
  "image_base64": "<base64-encoded JPEG or PNG>"
}
```
The base64 string may optionally include the `data:image/jpeg;base64,` data-URI prefix — it is stripped automatically.

**Success response (HTTP 200, JSON array):**
```json
[0.1234, -0.5678, 0.0123, ...]   // 128 floats
```

**Error responses:**
| HTTP | Reason |
|------|--------|
| 400  | Missing / empty `image_base64` field, or invalid base64 |
| 422  | No face detected in the image |
| 422  | Face embedding could not be computed |
| 500  | Unexpected server error |

---

### `GET /health`

Returns `{"status": "ok"}` — use for liveness/readiness probes.

## Spring Boot Configuration

Add/update `application.properties` (or the relevant profile):

```properties
# URL of the running face recognition microservice
face.recognition.service.url=http://localhost:5001

# Match threshold (cosine similarity, 0-1). 0.50 works well with 128-d embeddings.
# Lower value = more permissive (fewer false negatives, more false positives).
attendance.face.match.threshold=0.50
```

> **Important:** After enabling the external service, all previously onboarded employees must be **re-enrolled** (`POST /api/v1/admin/employees/onboard`) because the old embeddings were generated by the hash stub and are incompatible with real face embeddings.

## UI / API Notes — No Changes Required

The existing Flutter/web UI JSON payloads are correct and do not need to change:

**Onboarding** (`POST /api/v1/admin/employees/onboard`):
```json
{
  "employeeCode": "EMP001",
  "fullName": "John Doe",
  "department": "Engineering",
  "phone": "9999999999",
  "email": "john@example.com",
  "imagesBase64": [
    "data:image/jpeg;base64,/9j/4AAQ...",
    "data:image/jpeg;base64,/9j/4AAQ...",
    "data:image/jpeg;base64,/9j/4AAQ..."
  ]
}
```
- Provide **3 or more photos**: front-facing, slight left turn, slight right turn.  
- Each image should show only **one face**.  
- Good lighting, no sunglasses/mask.

**Entry** (`POST /api/v1/attendance/lodge-entry`):
```json
{
  "deviceId": "KIOSK-01",
  "imageBase64": "data:image/jpeg;base64,/9j/4AAQ..."
}
```

**Exit** (`POST /api/v1/attendance/mark-exit`):
```json
{
  "deviceId": "KIOSK-01",
  "imageBase64": "data:image/jpeg;base64,/9j/4AAQ..."
}
```

**Debug / Score Check** (`POST /api/v1/attendance/face-score`):
```json
{
  "deviceId": "KIOSK-01",
  "imageBase64": "data:image/jpeg;base64,/9j/4AAQ..."
}
```
Returns the best cosine similarity score **without** logging attendance — useful for diagnosing threshold issues from the UI.

## Tips for Best Accuracy

| Condition | Recommendation |
|-----------|---------------|
| Lighting  | Ensure even frontal lighting; avoid strong backlight |
| Distance  | Face should occupy > 30 % of the frame |
| Angle     | ± 30° yaw/pitch from frontal is tolerated |
| Image size | 300 × 300 px minimum; 640 × 480 px recommended |
| Format    | JPEG quality ≥ 70, or PNG |
| Obstructions | Remove sunglasses, hats, masks during enrolment and attendance |
